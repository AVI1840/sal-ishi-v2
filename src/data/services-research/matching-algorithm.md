# מנוע התאמה — 5 שכבות + דירוגים + למידה

## המטרה
להציג לאזרח ולמלווה את השירותים הכי מתאימים **מתוך 102 שירותים אמיתיים** — לא רק "מה יש", אלא "מה הכי טוב בשבילך".

## הערך
- **לאזרח:** "מישהו חשב בשבילי ומצא מה מתאים לי"
- **למלווה:** "לא צריכה לחפש ידנית — המערכת ממליצה"
- **למדינה:** 20% דחיית הדרדרות = ₪2 מיליארד חיסכון

---

## 5 שכבות ההתאמה

### שכבה 1: מניעת הדרדרות (40%)
מה מוכח מחקרית שמעכב ירידה תפקודית?
- שירות בקטגוריה "שייכות" או "בריאות" → ציון גבוה
- שירות שמתאים לדגלי סיכון של האזרח (בדידות, נפילות, קוגניציה)
- `match_score` מהקטלוג (0-100%) כבסיס

### שכבה 2: רצון ומוטיבציות (25%)
מה האזרח **רוצה** לעשות?
- התאמה בין `motivations` של האזרח ל-`subcategory` של השירות
- מוטיבציה "חברה" → מועדון חברתי
- מוטיבציה "בריאות" → התעמלות/בריכה
- מוטיבציה "משמעות" → התנדבות/לימוד

### שכבה 3: התאמה לפרופיל (20%)
מה האזרח **יכול** לעשות?
- `target_mobility` של השירות vs מצב תפקודי של האזרח
- שפה: `languages` של השירות כולל שפת האזרח?
- מגזר: שירות מתאים למגזר? (הפרדה, כשרות, שבת)

### שכבה 4: קרבה וזמינות (10%)
מה **נגיש** בפועל?
- שכונת השירות = שכונת האזרח → 100 נקודות
- שכונה סמוכה → 60 נקודות
- עירוני → 40 נקודות
- הסעה קיימת (`transport` in accessibility) → +20 בונוס

### שכבה 5: הוכחה חברתית — דירוגים (5%)
מה **אחרים** חושבים?
- ציון ממוצע מדירוגי אזרחים (1-5 כוכבים)
- כמות דירוגים (יותר = יותר אמין)
- דירוג מלווה (משקל כפול מאזרח)

---

## נוסחה

```
raw_score = (prevention × 0.40) + (motivation × 0.25) + (profile_fit × 0.20) + (proximity × 0.10) + (social_proof × 0.05)

persona_boost = 1.0 + persona_multiplier  // e.g., social_butterfly + group = +0.3

final_score = min(raw_score × persona_boost, 100)
```

## דירוגים — Schema

```json
{
  "service_id": "srv_007",
  "user_id": "citizen_001",
  "user_type": "citizen | coordinator",
  "rating": 4.5,
  "text": "מקום חם ונעים, אנשים מצוינים",
  "would_recommend": true,
  "date": "2026-06-01",
  "tags": ["חברה", "נעים", "נגיש"]
}
```

### השפעה על האלגוריתם:
- דירוג מלווה: משקל ×2 (מקצועי)
- דירוג אזרח: משקל ×1 (אישי)
- שירות עם < 3.5 ממוצע → ירידה ב-20% בציון
- שירות עם > 4.5 ממוצע → עלייה ב-10% בציון
- "would_recommend" = false מ-3+ אנשים → דגל לבדיקה

## Flow — מקצה לקצה

```
1. אזרח נקלט (אינטייק)
   → פרופיל: מוטיבציות, חסמים, מצב תפקודי, שכונה, שפה

2. מנוע התאמה רץ
   → 5 שכבות × 102 שירותים = ציון לכל שירות

3. מוצגות 5-8 המלצות מובילות
   → למלווה: עם הסבר "למה" לכל אחת
   → לאזרח: עם אמוג'ים, תמונה, "חינם/מסובסד"

4. אזרח בוחר → הזמנה

5. אחרי השירות → דירוג (1-5 כוכבים + טקסט)

6. דירוגים מעדכנים את שכבה 5

7. אייג'נט שבועי → מעדכן קטלוג + מתריע על שינויים
```
